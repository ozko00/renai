# renAI ユーザー仕様書

「24問・約3分で診断できる、あなただけの恋愛タイプを見つけるWebアプリ」のユーザー目線の仕様書。実装と1対1で対応する内容を、利用者が画面でできること・見えるもの・触れる前提で記述する。

---

## 1. プロダクト概要

### 1.1 プロダクト名
**renAI** — 16タイプの恋愛診断

### 1.2 一行コンセプト
24の問いに答えると、あなただけの恋愛タイプ（16種類のうちの1つ）と、AIで描き起こされたキャラクター画像が届く。

### 1.3 体験のコア
- 無料・登録不要・約3分で診断完了
- 結果は4軸（主導／受容、情熱／安定、言葉／行動、自由／一途）の組み合わせで決まる16タイプの中から1つ
- 自分のキャラクター画像をAIが毎回描き起こす（同じタイプでも軸の傾きで表情が変わる）
- 相性診断・シェアカード生成までワンセット

### 1.4 想定ユーザー
- スマホで気軽に診断したい10代後半〜30代の利用者
- SNSでシェアして友人・恋人・気になる人と話題にしたい層
- パートナー候補との相性を遊び感覚で確認したい層

### 1.5 注意書き
本診断はエンターテインメント目的のものです（フッターにも明示）。

```
[ホーム /]
  └─ 「診断をはじめる」

[一覧 /types]               (いつでもアクセス可)
[相性 /compatibility]
  └─ 相手のタイプを選択 → [相性結果 /compatibility/result]
```

  - いつでも中断OK・途中の答えは保存されます
  - 正解はない・直感で選んでください
- 補足: 「診断結果はあなたのブラウザにのみ保存されます」

### 3.3 質問画面 `/diagnosis`

24問の本体。1問ずつ表示し、選択するごとに自動で次へ進む。

**できること**
- 5段階の選択肢（とてもそう思う / そう思う / どちらともいえない / あまり思わない / 全く思わない）から1つ選ぶ
- 「←」で前の問題に戻り、選択を変更
- 「✕」で診断を中断（途中の回答は確認のうえ保存）
- 戻るボタンで再アクセスしたとき、回答済みの最後の続きから再開（`sessionStorage` に保存）

**見えるもの**
| 要素 | 内容 |
|------|------|
| ヘッダー | 「← / 1 / 24 / ✕」進捗カウンタ |
| プログレスバー | 横一直線、進むにつれて色が左から右に伸びる |
| 質問番号 | `QUESTION 01` 〜 `QUESTION 24` |
| 質問テキスト | 例: 「デートの計画は自分から提案したい」 |
| 5段階の丸ボタン | 中央が小さく、両端が大きい砂時計型レイアウト。選んだ瞬間に拡大して「✓ 選択しました」フィードバック |
| ヒント | 通常時「直感で選んでください」、選択直後「✓ 選択しました」 |

**振る舞い**
- 選択から約 0.5 秒後に自動で次の質問へ進む
- 連打しても二重遷移しない
- 24問目を選ぶと自動的に診断中画面へ遷移
- 中断時はブラウザ標準の confirm で確認、回答途中なら回答は保持される

**質問内容（24問・各軸6問）**

| 軸 | 内容 | 質問例 |
|---|---|---|
| LF（主導／受容） | 関係をリードしたいか相手のペースに合わせたいか | 「デートの計画は自分から提案したい」「気になる人には自分から話しかけにいく」 |
| PS（情熱／安定） | ドキドキを求めるか穏やかさを求めるか | 「恋はドキドキするものであってほしい」「一目惚れは、ある」 |
| WA（言葉／行動） | 気持ちを言葉で伝えるか行動で示すか | 「『好き』はちゃんと言葉で伝えたい」「長文のメッセージを送ることがある」 |
| IE（自由／一途） | 自分の世界を保ちたいか相手に深く向き合いたいか | 「恋人がいても自分の時間は大切にしたい」「束縛されるのは少し苦手だ」 |

### 3.4 診断中画面 `/diagnosis/loading`

AIが回答を分析している間に表示される待ち画面。

**できること**
- 待つ（自動で結果ページへ遷移）
- レート制限・通信エラー時は「もう一度試す」「質問に戻る」を選べる

**見えるもの**
- 通常: 「あなたの恋ごころを 読み解いています」+「. . .」のドットアニメーション
- レート制限時: 「すこしだけお時間をください」+ 残り秒数表示
- エラー時: 「うまくいきませんでした」+ エラー内容

**振る舞い**
- 内部的に診断API（`/api/diagnosis`）を呼び出し、続けてキャラクター画像API（`/api/character`）をウォームアップ
- 成功すると診断結果を `localStorage` に保存して `/diagnosis/result` へ自動遷移
- 回答が `sessionStorage` にない場合は質問画面へ戻される
- レート制限は1分間に5回まで（IPベース）

### 3.5 結果画面 `/diagnosis/result`

診断の中核となる結果表示。

**できること**
- 結果のキャラクター画像と概要を見る
- 「相性のいいタイプを見る」で相性診断へ
- 「シェアカードを作る」でシェアカード画面へ
- 「16タイプを見る」で一覧へ
- 「もう一度診断する」で診断ガイドへ
- 戻る矢印でトップへ

**見えるもの**
| セクション | 内容 |
|------|------|
| キャラクターカード | フルサイズのAI生成キャラ画像。下部にグラデーションマスクで `YOUR TYPE · LPWI` 等のコード、タイプ名（例「太陽の恋人」）、ショートコピー（例「輝いて誘う人」）を重ねる |
| サマリー | AIが生成した数行の総評（軸スコアの傾向を踏まえた個別文章） |
| 相性CTA（上） | 「相性のいいタイプを見る」リンクカード |
| 01 · AXES | 4軸スコアの可視化バー。例: LF=0.8 なら「主導 90% / 受け身 10%」 |
| 02 · ADVICE | AIが生成したあなた向けアドバイス3〜5項目（番号付き） |
| 相性CTA（下） | グラデーション背景の大きなCTA「相性のいい人を見つけにいきましょう」 |
| 下部アクション | 「シェアカードを作る」「16タイプを見る」「もう一度診断する」 |

**振る舞い**
- `localStorage` に結果がなければ自動で `/diagnosis/start` へ戻される
- 旧スキーマ（v1）の結果はその場で v2 へ自動マイグレーションされる
- AIが軸スコア生値（例: `(PS:-0.17)`）をうっかり混入させた場合、表示時に除去

### 3.6 シェアカード画面 `/diagnosis/share`

SNS共有・画像保存に最適化されたカード。

**できること**
- 「画像として保存」でPNGダウンロード（または Web Share API でシェア）
- Twitter/X / LINE のシェアボタンでテキスト付き共有
- 戻る矢印で結果画面へ戻る

**見えるもの**
- 4:5 比率のシェアカード本体
  - 背景: 淡いブルーのグラデーション
  - キャラクター画像をフルサイズで配置
  - 上部左: `● renAI` ロゴ／上部右: タイプコード（例 `LPWI`）
  - 下部中央: 「あなたの恋愛タイプは」+ タイプ名 + ショートコピー
  - 下部左: 年月（例 `2026.04`）／下部右: `renAI.app`
- 保存ボタン
- Twitter/X・LINE共有ボタン

**振る舞い**
- カードのDOMを `html-to-image` で2倍解像度のPNGに書き出し
- ファイル名は `renAI_LPWE_2026-04.png` のようにタイプコード入り
- iOS等では Web Share API 優先、未対応環境では `<a download>` でフォールバック
- 保存成功・失敗は2.5秒のトーストで通知
- シェアテキスト例: 「わたしの恋ごころは『太陽の恋人 🌞』でした。 輝いて誘う人 #renAI」

### 3.8 相性診断画面 `/compatibility`

自分のタイプと相手のタイプから相性を見る。

**できること**
- 自分の最新の診断結果が自動で表示される
- 16タイプの中から相手のタイプを選ぶ（ラジオ選択）
- 「相性を診断する」で相性結果画面へ進む
- 自分の診断結果がない場合は「診断をはじめる」誘導

**見えるもの**
- 「ふたりの恋ごころ、重ねてみませんか」見出し
- `01 · あなた` セクション: 絵文字＋コード＋タイプ名
- `02 · 気になる人` セクション: 16タイプを2〜3列のグリッドで表示。選択中のカードは枠強調
- 補足: 「相手にも診断を受けてもらうのが一番ですが、予想で選んでもOK」

### 3.9 相性結果画面 `/compatibility/result`

軸別の相性を点数と読み解きで見る。

**できること**
- 相性スコア（0〜100）を確認
- 4軸ごとの一致／対比をMATCH／CONTRASTバッジ付きで確認
- 「別のタイプと診断する」で相性入力画面へ戻る
- 「自分の結果に戻る」で結果画面へ
- 戻る矢印で相性入力画面へ

**見えるもの**
| セクション | 内容 |
|---|---|
| スコアヒーロー | `64 / 100` 等の大きな数字＋一行コメント |
| 自分・相手のチップ | 絵文字＋コード＋タイプ名を並列表示 |
| 01 · OVERVIEW | 「ふたりの傾き」総評 |
| 02 · AXES | 4軸ごとに `LF MATCH` か `CONTRAST` バッジ＋自分側ラベル × 相手側ラベル＋一文コメント |

**算出ルール（ユーザーへの説明レベル）**
- 4軸それぞれが一致するほど高得点
- 一致＝MATCH、不一致＝CONTRAST と表示
- 不一致でも「補い合う関係」として読み解きが添えられる

## 4. キャラクター画像の仕様

### 4.1 体験
- 結果画面・シェアカードに、自分のタイプに対応した動物キャラクターの画像が表示される
- 画像はAI（Gemini）でリアルタイム生成
- 同じタイプでも、4軸の傾き具合で表情やポーズが微妙に変わる

### 4.2 待ち時間と失敗時のフォールバック
- 通常はローディング画面で先行ウォームアップされ、結果画面到達時にはほぼ表示準備完了
- 生成失敗時は絵文字（タイプの代表絵文字）が代替表示される — 結果が見られなくなることはない
- 同じパラメータならキャッシュから即時返却（24時間 + 7日間 SWR）

## 5. データの保存と取り扱い

### 5.1 ブラウザに保存される情報
| キー | 場所 | 用途 |
|---|---|---|
| `renAIAnswers` | `sessionStorage` | 24問の回答途中データ。診断完了で削除 |
| `diagnosisResult` | `localStorage` | 直近の診断結果。次回起動時に結果画面・相性診断で再利用 |
| `compatibilityRequest` | `localStorage` | 相性診断のために選んだ自分／相手コードのペア |

### 5.2 サーバーへ送信される情報
- 24問の回答（無記名、IPベースの簡易レート制限のみ）
- 診断・キャラクター生成時にAIへ送るプロンプトには軸スコアとタイプコードのみが含まれる
- 名前・メール・端末ID等の個人特定情報は一切収集しない

### 5.3 利用制限
- 1分間に5回まで診断 / キャラクター生成可能（IP単位）
- 上限到達時はローディング画面で「すこしだけお時間をください」表示と再試行ガイダンス

## 6. UI/UXの約束

### 6.1 デザイントーン
- カラーパレット: 淡いスカイブルー基調（`--koi-bg` 系）、アクセントは桃色〜サーモン
- タイポ: 日本語は明朝（serif-jp）でエディトリアルな空気感、コードは等幅フォント
- 装飾: 角の丸い大きなカード、淡いブロブ、控えめなスパークルアイコン

### 6.2 アクセシビリティ
- すべての画像装飾要素は `aria-hidden`、コンテンツ画像には `alt` テキスト
- 質問の選択肢は `role="radiogroup"`/`role="radio"` でキーボード対応
- フォーカスリング表示
- 進行状況・状態変化は `aria-live="polite"` で通知

### 6.3 レスポンシブ
- モバイル優先（`max-w-md` 幅）の縦スクロール体験
- タブレット・デスクトップでも `max-w-2xl` までで読みやすさを担保

### 6.4 中断耐性
- 質問途中でブラウザを閉じても、同じセッションなら最後の続きから再開
- 結果は `localStorage` に保存され、再訪時に即座に表示

## 7. エラー・例外時の振る舞い

| 状況 | ユーザーが見るもの |
|---|---|
| 診断回答が破損 | 自動で質問画面へ戻され、回答途中なら継続可能 |
| AIサーバーエラー | 「うまくいきませんでした」＋詳細＋「もう一度試す」「質問に戻る」 |
| レート制限超過 | 「すこしだけお時間をください」＋残り秒数＋再試行ボタン |
| API キー不正（管理者向け） | 「管理者にお問い合わせください」表示 |
| キャラクター画像生成失敗 | 絵文字フォールバックで結果は表示続行（致命的ではない） |
| 結果が `localStorage` に存在しない | 自動的に診断ガイド画面へリダイレクト |
| 旧スキーマの結果が残っている | 自動マイグレーションして表示続行（再診断不要） |

## 9. 用語集

| 用語 | 意味 |
|---|---|
| 4軸 | LF（主導/受容）、PS（情熱/安定）、WA（言葉/行動）、IE（自由/一途）の4つの恋愛行動軸 |
| タイプコード | `LPWI` のような4文字コード。各軸の前者(L/P/W/I) or 後者(F/S/A/E)の組み合わせ |
| 16タイプ | 4軸×2方向の組み合わせで生まれる16通りの恋愛タイプ |
| ショートコピー | 「輝いて誘う人」のようにタイプを一言で表すコピー |
| トーンカラー | 各タイプに割り当てられた象徴色 |
| 恋ごころ | 本プロダクトでユーザーの恋愛傾向を指す総称 |

## 10. 仕様外（明示的に対象外）

- ユーザー登録・アカウント機能
- 履歴の長期保存（同じブラウザ内 `localStorage` のみ）
- 通知機能・メール配信
- 第三者へのデータ提供
- 医療・心理学的な診断としての利用（エンタメ目的のみ）
